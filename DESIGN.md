# ドライブレコーダー動画合成スクリプト 設計書

## 概要
ドライブレコーダーで撮影された前方カメラと後方カメラの2つの動画ファイルを、ピクチャーインピクチャー形式で1つの動画に合成するmacOS用Pythonスクリプト。

## 要件

### 機能要件
1. **個別ファイル処理モード**: 2つの動画ファイル（前方カメラ、後方カメラ）を引数で受け取り合成
2. **バッチ処理モード**: フォルダを指定して、配下の前方/後方カメラのペアを自動検出して一括合成
3. ffmpegを使用してピクチャーインピクチャー形式で合成
4. 出力ファイル名は入力ファイル名（前方カメラ）の拡張子前に `_overlay` を追加
5. 出力先は入力ファイルと同じディレクトリ

### 技術要件
- 実行環境: macOS
- スクリプト言語: **Python 3.x**
- 依存ツール: ffmpeg (Homebrewなどでインストール済みを想定)
- 必要なPythonライブラリ: 標準ライブラリのみ（os, sys, subprocess, pathlib, re, datetime）

## スクリプト仕様

### ファイル名
`overlay.py`

### 実行方法

スクリプトに実行権限を付与して直接実行可能：
```bash
chmod +x overlay.py
```

#### 個別ファイル処理モード
```bash
./overlay.py <前方カメラ動画> <後方カメラ動画>
```

#### バッチ処理モード
```bash
./overlay.py <フォルダパス>
```

### 引数
- **2つの引数を指定した場合（個別ファイル処理モード）**:
  - 第1引数: 前方カメラの動画ファイルパス（必須）
  - 第2引数: 後方カメラの動画ファイルパス（必須）

- **1つの引数を指定した場合（バッチ処理モード）**:
  - 第1引数: 動画ファイルが格納されているフォルダパス（必須）

### ドライブレコーダーファイル名形式

ドライブレコーダーの動画ファイルは以下の形式で命名されています：

**形式**: `yymmddHHMMss_[種別][カメラ向き].MP4`

- **yymmdd**: 年(下2桁)月日
- **HHMMSS**: 時分秒
- **種別**: 録画の種別を示す英大文字
  - `E`: 衝撃イベント録画
  - `U`: ユーザー手動録画
  - など
- **カメラ向き**: カメラの向きを示す英大文字
  - `F`: 前方カメラ (Front)
  - `B`: 後方カメラ (Back)

**例**:
- `251008125706_EF.MP4`: 2025年10月8日12時57分06秒の衝撃イベント前方カメラ映像
- `251008125706_EB.MP4`: 同時刻の衝撃イベント後方カメラ映像
- `250923162018_UF.MP4`: 2025年9月23日16時20分18秒のユーザー手動録画前方カメラ映像

### バッチ処理モードの動作

1. 指定されたフォルダ内の全ての `.MP4` ファイルを走査
2. ファイル名から日時、種別、カメラ向きを抽出
3. 同じ日時（±1秒以内）の前方カメラ(F)と後方カメラ(B)のペアを検出
4. 検出されたペアごとに合成処理を実行
5. 既に `_overlay.MP4` ファイルが存在する場合はスキップ

**ペアリングルール**:
- 日時が完全一致するものを優先
- 1秒のずれがある場合も同じタイミングとして扱う
- 同じ種別（E、Uなど）のファイル同士をペアにする

### 出力

#### 出力ファイル名の生成ルール

**ドライブレコーダーファイル形式の場合** (`yymmddHHMMss_[種別][カメラ向き].MP4`):
- カメラ向き（F/B）を削除し、種別のみを残して`_overlay.MP4`を追加
- 例:
  - 入力: `251008125706_EF.MP4` / `251008125706_EB.MP4`
  - 出力: `251008125706_E_overlay.MP4`
  - 入力: `250923162018_UF.MP4` / `250923162018_UB.MP4`
  - 出力: `250923162018_U_overlay.MP4`

**一般的なファイル形式の場合**:
- 第1引数（前方カメラ）のファイル名の拡張子前に`_overlay`を追加
- 例:
  - 入力: `video.MP4`
  - 出力: `video_overlay.MP4`

#### 出力ファイルのタイムスタンプ設定

- **ドライブレコーダーファイル形式の場合**: 出力ファイルのタイムスタンプを、前方カメラファイルから解析した日時に設定します
  - 例: `251008125706_EF.MP4` から作成されたファイルは、2025年10月8日12時57分06秒のタイムスタンプが設定されます
- **一般的なファイル形式の場合**: ファイルを作成した時刻のタイムスタンプを保持します

出力先は入力ファイルと同じディレクトリ

### ffmpegコマンド仕様

```bash
ffmpeg -i <前方カメラ動画> -i <後方カメラ動画> \
  -filter_complex "[0:v]crop=iw:ih*0.9[v0];[1:v]scale=iw/2.5:ih/2.5,crop=iw:ih*0.66[v1];[v0][v1]overlay=W-w-20:H-h-70[v]" \
  -map "[v]" \
  -map 0:a \
  -c:v libx264 \
  -preset veryfast \
  -crf 23 \
  -c:a copy \
  -shortest \
  <出力ファイル>
```

#### フィルター詳細
1. **前方カメラ処理** `[0:v]crop=iw:ih*0.9[v0]`
   - 上下を90%にクロップ（上下10%をカット）

2. **後方カメラ処理** `[1:v]scale=iw/2.5:ih/2.5,crop=iw:ih*0.66[v1]`
   - サイズを2.5分の1に縮小
   - 上下を66%にクロップ（上下34%をカット）

3. **オーバーレイ** `[v0][v1]overlay=W-w-20:H-h-70[v]`
   - 後方カメラ映像を前方カメラ映像の右下に配置
   - 右端から20ピクセル、下端から70ピクセルの位置

4. **音声処理**
   - `-map 0:a`: 前方カメラ（第1入力）の音声トラックを使用
   - `-c:a copy`: 音声を再エンコードせずにコピー（高速処理）

5. **エンコード設定**
   - コーデック: libx264
   - プリセット: veryfast（高速エンコード）
   - CRF: 23（品質設定）
   - `-shortest`: 短い方の動画に合わせて終了

## エラーハンドリング

### チェック項目
1. 引数の数が1つまたは2つであることを確認
2. 個別ファイル処理モード時:
   - 第1引数のファイルが存在することを確認
   - 第2引数のファイルが存在することを確認
3. バッチ処理モード時:
   - 第1引数がディレクトリであることを確認
   - ディレクトリ内に処理対象のMP4ファイルが存在することを確認
4. ffmpegコマンドが利用可能であることを確認
5. 出力ファイルが既に存在する場合はスキップ（メッセージ表示）

### エラーメッセージ
- 引数なし: "Usage: ./overlay.py <front_camera_video> <rear_camera_video> or ./overlay.py <folder_path>"
- 引数不足: "使用方法: ./overlay.py <前方カメラ動画> <後方カメラ動画> または ./overlay.py <フォルダパス>"
- ファイル不存在: "エラー: ファイルが見つかりません: <ファイル名>"
- ディレクトリ不存在: "エラー: ディレクトリが見つかりません: <ディレクトリ名>"
- ffmpeg未インストール: "Error: ffmpeg not found. Please install with: brew install ffmpeg"
- ペアが見つからない: "警告: ペアとなる後方カメラ映像が見つかりません: <ファイル名>"
- タイムスタンプ設定失敗: "警告: タイムスタンプ設定に失敗しました: <エラーメッセージ>"

## 実装の流れ

### 共通処理
1. シバン（shebang）の設定: `#!/usr/bin/env python3`
2. 必要なライブラリのインポート
3. ffmpegコマンドの存在確認関数
4. ffmpeg実行関数の実装

### 個別ファイル処理モード
1. 引数が2つであることを確認
2. 両ファイルの存在確認
3. 出力ファイル名の生成
4. ffmpegコマンドの実行
5. 完了メッセージの表示

### バッチ処理モード
1. 引数が1つでディレクトリであることを確認
2. ディレクトリ内のMP4ファイルを走査
3. ファイル名から日時・種別・カメラ向きを抽出するパース関数の実装
4. 前方カメラファイルのリストを作成
5. 各前方カメラファイルに対して：
   a. 対応する後方カメラファイル（±1秒以内、同種別）を検索
   b. ペアが見つかった場合、出力ファイルの存在確認
   c. 出力ファイルが存在しない場合、ffmpegで合成処理
   d. 処理結果のログ出力
6. バッチ処理完了サマリーの表示

### メイン関数
1. コマンドライン引数の数をチェック
2. 引数の数に応じて個別ファイル処理モードまたはバッチ処理モードを実行
3. 例外処理とエラーメッセージの表示

## 補足事項

- **初回実行前に実行権限を付与**: `chmod +x overlay.py`
- **実行方法**: `./overlay.py` として直接実行可能
- ffmpegのインストール方法: `brew install ffmpeg`
- 処理時間は動画の長さとマシンスペックに依存する
- バッチ処理時は複数の動画を順次処理するため、時間がかかる場合がある
- Python 3.6以上を推奨（pathlibとf-stringsを使用）
- 音声は前方カメラの音声トラックが使用される
- 再生速度の変更は行わない（元の速度を維持）

## モジュール設計

### 主要な関数

1. `check_ffmpeg()`: ffmpegの存在確認
2. `parse_filename(filename)`: ファイル名から日時、種別、カメラ向きを抽出
3. `generate_output_filename(front_file)`: 出力ファイル名を生成（ドライブレコーダー形式対応）
4. `set_file_timestamp(file_path, timestamp)`: ファイルのタイムスタンプを設定（新規）
5. `find_pair(front_file, files_list)`: 前方カメラファイルに対応する後方カメラファイルを検索
6. `run_ffmpeg(front_file, back_file, output_file)`: ffmpegコマンドを実行して合成（タイムスタンプ設定機能付き）
7. `process_single_pair(front_file, back_file)`: 個別ファイル処理モード
8. `process_batch(directory)`: バッチ処理モード
9. `main()`: メイン処理（引数解析と各モードへの振り分け）
