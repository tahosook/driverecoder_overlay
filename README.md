# ドライブレコーダー動画合成スクリプト

ドライブレコーダーで撮影された前方カメラと後方カメラの2つの動画ファイルを、ピクチャーインピクチャー形式で1つの動画に合成するPythonスクリプトです。

## 特徴

- **個別ファイル処理モード**: 2つの動画ファイルを個別に指定して処理可能
- **バッチ処理モード**: フォルダ内のファイルを自動検出して一括処理可能
- **ドライブレコーダー対応**: 特定の命名規則に基づいた自動ペアリング機能
- **タイムスタンプ保持**: 出力ファイルにオリジナルファイルの日時情報が設定されます
- **高音質**: 音声は前方カメラのソースを利用（再エンコードなし）

## 要件

- **Python**: 3.6以上
- **ffmpeg**: Homebrewでインストール
  ```bash
  brew install ffmpeg
  ```

## インストール

1. リポジトリをクローンまたはダウンロード
2. 実行権限を付与
   ```bash
   chmod +x overlay.py
   ```

## 使用方法

### 個別ファイル処理モード

2つの動画ファイルを個別に指定して合成する場合：

```bash
./overlay.py video1.MP4 video2.MP4
```

### バッチ処理モード

フォルダを指定して、同フォルダ内の動画ファイルを自動検出・処理する場合：

```bash
./overlay.py /path/to/video/folder
```

## ファイル命名規則

### 入力ファイル形式（ドライブレコーダー）

```
yymmddHHMMss_[種別][カメラ向き].MP4
```

- `yymmdd`: 年(下2桁)月日
- `HHMMSS`: 時分秒
- `種別`: 録画の種別（E: 衝撃イベント, U: ユーザー手動録画など）
- `カメラ向き`: F(前方), B(後方)

**例**:
- `251008125706_EF.MP4`: 2025年10月8日 12:57:06 前方カメラ 衝撃イベント
- `251008125706_EB.MP4`: 2025年10月8日 12:57:06 後方カメラ 衝撃イベント

### 出力ファイル形式

**ドライブレコーダー形式の場合**:
- 入力: `251008125706_EF.MP4` + `251008125706_EB.MP4`
- 出力: `251008125706_E_overlay.MP4`

**一般形式の場合**:
- 入力: `video.MP4`
- 出力: `video_overlay.MP4`

### バッチ処理のペアリングルール

1. ファイル名から日時・種別・カメラ向きを解析
2. 同じ日時（±1秒以内）かつ同じ種別で、前方向(F)と後方向(B)のペアを作成
3. 誤認識防止のため、1秒以上のずれがある場合は別ファイルとして扱う

## ビデオ合成仕様

### レイアウト
- 前方カメラ動画をメインに表示
- 後方カメラ動画を右下隅にピクチャーインピクチャーで表示
- 位置: 右端から20px、下端から70px

### 映像処理
- **前方カメラ**: 上下10%クロップ
- **後方カメラ**: 65%縮小 + 上下34%クロップ
- **エンコーディング**: H.264, CRF=23, 高速プリセット

### 音声処理
- 前方カメラの音声をそのまま使用（再エンコードなし）
- 短い方の動画長に合わせて終了

## タイムスタンプ設定

出力ファイルのタイムスタンプは以下の規則で設定されます：

- **ドライブレコーダー形式**: 前方カメラファイル名から解析した日時に設定
- **一般形式**: ファイル作成時のタイムスタンプを使用

## エラーハンドリング

### 引数チェック
- 引数なしの場合: 使用方法を表示
- 引数が多すぎる場合: エラーメッセージを表示

### ファイルチェック
- 入力ファイルが存在しない場合: エラー表示
- 出力ファイルが既に存在する場合: スキップ

### ffmpegチェック
- ffmpegがインストールされていない場合: エラーメッセージとインストール方法を表示

## 制限事項

- ドライブレコーダーの命名規則にのみ対応
- バッチ処理では完全自動判定のため、誤判定が発生する可能性あり
- 大きな動画ファイルの場合は処理に時間がかかる

## Q&A

**Q: ffmpegがインストールされていません**  
A: Homebrewでインストールしてください: `brew install ffmpeg`

**Q: タイムスタンプが設定されません**  
A: 権限の問題かファイルシステムの制限の可能性があります。通常動作には影響ありません。

**Q: バッチ処理でペアが見つからない**  
A: ファイル名が正しい形式かを確認してください。1秒以上の時間ずれがある場合は別ファイルとして扱われます。
